# Rocco App

## Getting Started

## Prerequisites

- Flutter version: 3.35.2
- Dart version: 3.9.0


## Installing

1. Clone the repository
2. Navigate to the project directory
3. Install dependencies

   ```
   flutter pub get
   ```

## Run the project

**Development Debug**

```
flutter run --flavor development -t lib/main.dart --dart-define-from-file=config.development.json


**Development Release**

```
flutter run --flavor development -t lib/main.dart --dart-define-from-file=config.development.json --release

```

**Production Debug**

```
flutter run --flavor production -t lib/main.dart --dart-define-from-file=config.Production.json


```

**Production Release**

```
flutter run --flavor production -t lib/main.dart --dart-define-from-file=config.production.json --release

```

### core

> Contains core functionalities and shared resources used across the entire application.

- **configs:** Contains configuration files and settings used in the project.
- **cubits:** Houses the Cubit classes responsible for managing the application's state.
- **data:** Handles data-related operations, including local data storage, models, network requests, and repositories.
  - **local:** Manages local data storage, such as databases or shared preferences.
  - **network:** Handles network-related operations, such as API requests and responses.
  - **models:** Defines data models used throughout the application.
  - **repositories:** Implements data access logic and serves as a bridge between the data sources and the rest of the application.
- **routes:** Navigation routes and screen definitions.
- **services:** Contains service classes responsible for handling various functionalities, such as event handling, Firebase integration, in-app purchases, etc.
- **shared:** Shared resources such as constants, themes, utilities, and widgets.

### generated

> Contains auto-generated files, such as code generated by build tools.

### modules

> Contains feature modules that encapsulate specific functionalities of the application.

- **auth:** Authentication feature module.
- **dashboard:** Dashboard feature module.
- **...:** Additional feature modules as needed.

## Usage

1. To generate code runner
   [Build Runner](https://pub.dev/packages/build_runner)

   ```
   dart run build_runner build --delete-conflicting-outputs

  

2. To generate localization
   [Easy Localization](https://pub.dev/packages/easy_localization)

   ```
   dart run easy_localization:generate -S assets/translations -O lib/generated -f keys -o locale_keys.dart
   ```

3. To add Firebase files
   - Download google-service.json / GoogleService-Info.plist on the Firebase console
   - Add google-service.json / GoogleService-Info.plist to **assets/firebase** (dev,staging, production) folders
   - Run command
     ```

# for adding firebase in flavour run: 

     dart run flutter_flavorizr -p google:firebase
     

# for adding android confing run this      
     ```
    dart run flutter_flavorizr -p android:buildGradle,android:flavorizrGradle,android:androidManifest

# for adding ios and other config run : 
   
   ```
    dart run flutter_flavorizr

## Build realese

### Android

**Development Flavor**

```
flutter build apk --flavor development --release -t lib/main.dart --dart-define-from-file=config.development.json

use if fvm :
fvm flutter build apk --flavor development --release -t lib/main.dart --dart-define-from-file=config.development.json

```

**Production Flavor - APK**

```
flutter build apk --flavor production --release -t lib/main.dart --dart-define-from-file=config.production.json


use if fvm :
flutter build apk --flavor production --release -t lib/main.dart --dart-define-from-file=config.production.json

```

**Production Flavor - AppBundle**

```
flutter build appbundle --flavor production --release -t lib/main.dart --dart-define-from-file=config.production.json


use if fvm :

```
